Come from my another small local editing tools - notes, which is a windows software.
So the requirements are more: write anything easily and safely on any OS.
And the architecture is evolved: The notes architecture "local small web server + software with embed browser" is evolved to "browser + local small web server synced with internet"

Architecture:
User can use oe web site on internet, or without network connection user can use local web server and sync with internet later
local: a ruby on rails web app running locally, which will sync with internet web site timely. server side: The ruby on rails oe server

Tech features:
1. Integrate openID.
2. Use fckeditor as online editor
3. Auto-save timely
4. Easy to publish to lots of blog or export as archive

open-open platform features:
I create the open-open platform concept, to allow any kind of rubyonrails web site become a open-platform, just by download/install my open-open platform plugin. Then other web site can use the standard open API of your web site to access the service you provide. Is it cool?
So actually, the oe.joyqom.com web site is a website which just use the API of my another web app "oe".
