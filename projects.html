Project

 

E-Business Service Applciation Platform(PS)

Summary

A Middle layer System for e-business application, which provides the application logic implementation platform, the Data Access layer and IDE(Integrated Dev Environment). It mainly includes the Script Compiler/Virtual Machine, Data Object Access Engine and the Communication layer(socket). Users can write c-style scripts to implement the application logic more easily and use the Data Object Access Engine to operate data more conveniently on variable kinds of DB, even FS. XML is used to define data structure.The communication layer implements the fast remote call by socket. The IDE is used to define data structure in visual way(UI), and make it easy to write and debug the script. The whole platform including IDE is OS-independent and DB-independent.

Develop Tools

VC(for win32), GCC(for linux), XLC(for AIX)

Platform and Architecture

Support Linux ，Windows,  AIX, PowerPC

DataBase

Support IBM DB2, ORACLE, MS SQL SERVER, DBF, mySQL

Develop Language

C++, java，Asm(Intel x86, IBM RS6000, PowerPC)

Time

2001.1 – 2001.10

Role

Architect & Main Programmer，do system analyzing, designing and coding.

Responsibility

1.     Design and Implement Scripte Engine(compiler and VM)

2.     Use java to develop the “PS Studio”, the “Development Environment” for the Script, which provides the platform for user to edit, compile and run the scripts.

3.     Use java to write the PS client for java application.

4.     Use VC to write the PS client COM for ASP

5.     Use MFC+COM to write the Adiministrator Tools, which inlucdes a xml editor(because the server config is stored in XML format file).

6.     Use java to write the function test tool for PS server, which will call the java client for PS)

7.     Use VC to write the stress test tool, performance test tool and stability test tool(based on GUI). These tools will call the PS client COM.
